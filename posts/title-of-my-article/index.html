1 YAML header format

Reproduced here with comments, but it’s also included as part of this file which you can copy-paste inside your new article. Must appear at the very beginning of the file and enclosed with triple hyphens.

---
title: Title of my article # use double-quotes if you need to escape characters
pagetitle: Slightly different title # optional, defaults to title, appears in the browser title bar
author: 
- alix
- my alter ego
- the demon on my shoulder
date: 2026-02-01
updated: 2026-03-03 # optional
abstract: | # optional, will be rendered at the top of the page just under the title
  Lorem ipsum and so forth
note: | # optional, html will be injected as-is after the title and abstract but before the ToC and the main text
  <h2>Editor's Note</h2>
  <p>Here be dragons, <a href="https://weather.com/retro/">return to safety</a>.</p>
show-toc: true # optional, defaults to false, render table of content,
number-sections: true # optional, defaults to false, whether to number sections 1, 2, 3, subsections 1.1, 1.2, 2.1, does not go deeper than subsections
custom-css: # optional, for article-specific CSS and avoid polluting the common stylesheet /static/css/index.css
- bespoke.css
- pizzazz.css
custom-js: # optional, same concept as custom-css, but for scripts
- magic.js
- arcane.js
---

2 How to create a new post

  • Create a sub-folder in ./md/posts/ with a simple kebab-case name based on the title. For example for this template this would be /md/posts/title-of-my-article/. The name can/should be shortened if the title is too long.
  • Create an index.md file inside the folder containing the above YAML block. You can find a version without comments (don’t leave any in the YAML block) in the source file /md/posts/title-of-my-article/index.md.
  • When you run the build.sh script calls the /src/render.sh script once for each subfolder of /md/posts/. Pandoc renders index.md into index.html and moves it to /output/posts/title-of-my-article/index.html.
  • Note that you can create additional subfolders with their own index.md if your post is split into more than more page, those will be rendered as well. The build.sh scripts simply recursively traverses all subfolder inside /md/posts/ and calls render.sh on every index.md it encounters.

3 “My post uses images and other somesuches”, you inquire

Worry not,

  • Create a subfolder with the same name in the static asset folder and put all your assets inside it. In our case this would be /static/posts/title-of-my-article/.
  • All files and subdirectories you put in that folder will end up in the same folder alongside index.html in the final output.

4 “But alix”, I hear you say, “my post also uses many fancy citations so I need a bibliography”

Sure thing, we love a good bibliography,

  • When the render script detects the presence of the bibtex file /md/posts/title-of-my-article/references.bib, it calls pandoc with the --citeproc option which will automatically render a Bibliography section after the main text, but before the footnotes with nice links and a citation tooltip function and so forth.
  • In your main text you can add citations with citekey matching those in references.bib such as
    • [@foo_bar_2002]
    • [@foo_bar_2002; @bork_bark_2035]
    • [-@foo_bar_2002] (only include the year in parentheses)
  • Note that WHSAH uses APA style. Let’s self cite, as is tradition (alix et al., 2026).
  • See https://pandoc.org/demo/example33/8.20-citation-syntax.html#citation-syntax for additional options.

5 “But aaliiiix”, I hear you whine, “my article uses fancy CSS with cool interactive buttons, what do?”

Put all your css files in the static asset folder of your article and list them under the custom-css tag in the YAML header. Same for javascript files, list them under the custom-js tag. In our case, this will tell pandoc to include the following in the html header:

<head>
    <!-- ... -->
    <link rel="stylesheet" href="bespoke.css" />
    <link rel="stylesheet" href="pizzazz.css" />
    <script type="module" src="magic.js"></script>
    <script type="module" src="arcane.js"></script>
    <!-- ... -->
</head>

Make sure all your stylesheets and scripts are located in the static asset folder /static/posts/title-of-my-article/.
